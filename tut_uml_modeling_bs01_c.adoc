== 準備

[.lead]
このチュートリアルでは、モデリングツールとして {astah} Professional を、実装にはC言語を使います。
最初に、演習に必要な環境を用意しましょう。

=== 実装環境を用意する

このチュートリアルでは、実装に C言語 を使います。Windows、Mac、Linuxで動作するプログラムになるよう、C99標準に準拠したできるだけポータブルなコードを作成しようと思います。

それぞれの環境でのC言語用コンパイラのセットアップ方法を紹介しておきます。

==== Windows環境

Windowsでは、以下のいずれかの方法でCコンパイラを用意できます：

MinGW-w64を利用する場合::
* MinGW-w64のインストーラーをダウンロード: `https://www.mingw-w64.org/`
* インストール後、環境変数PATHにMinGW-w64のbinディレクトリを追加

NOTE: ほぼ同じ環境として、MSYS2をインストールする方法あります。

Visual Studio Build Toolsを利用する場合::
* Visual Studio Community（無料版）をインストール: `https://visualstudio.microsoft.com/ja/`
* インストール時に「C++によるデスクトップ開発」を選択

WSL2とUbuntuを利用する場合::
* PowerShellを管理者で実行し、 `wsl --install` を実行後、PCを再起動する
* Ubuntuのターミナルを起動し、`sudo apt update upgrade` を実行
* `apt install gcc` を実行

==== macOS環境

macOSでは、Xcodeのコマンドラインツールをインストールします：

[source,console]
----
xcode-select --install
----

これにより、gccコマンド（実際はclangへのエイリアス）が使用可能になります。

==== Linux環境

多くのLinuxディストリビューションでは、gccがデフォルトでインストールされています。
インストールされていない場合は、パッケージマネージャーでインストールします：

Ubuntu/Debian::
[source,console]
----
sudo apt-get update
sudo apt-get install build-essential
----

Fedora/RHEL::
[source,console]
----
sudo dnf install gcc make
----

==== コンパイラの動作確認

正しくインストールされたか確認するには、次のコマンドを実行します：

[source,console]
----
gcc --version
----

または

[source,console]
----
cc --version
----

バージョン情報が表示されれば、正常にインストールされています。

==== C言語の基礎知識

このチュートリアルでは、C言語の高度な機能は使いませんが、基本的な知識は必要です。
C言語になじみがない方は、以下のような参考資料を確認することをお勧めします：

* 『作って身につくC言語入門』（久保秋 真 著） <<kuboaki01>>
* 『新・明解C言語 入門編』（柴田 望洋 著） <<sibata01>>
* オンライン学習サイト: `https://www.learn-c.org/` <<learnc>>

このチュートリアルで使用する主なC言語の機能：

* 構造体 (`struct`)
* 列挙型 (`enum`)
* 関数とポインタ
* 標準ライブラリ (`stdio.h`, `stdlib.h`, `string.h`, `time.h`, `stdbool.h`)

複数の図で表すのは、大きな問題ならば分割して小さな問題として捉えるほうが整理しやすいからです。
たとえば、システム全体の構造と、その内部のある構成要素の内部の構造を別の図に分けることができるでしょう。
また、関心事によって表すもの（モデルに残すものや削るもの）が変わるからです。
たとえば、あるシステムを設計するのに、構造を表すためにクラス図を使い、振舞いを表すのにシーケンス図やステートマシン図使うといったことです。

=== ボウリングのスコアのつけ方

このチュートリアルで作成するのは、ボウリングのスコアを記録し、合計スコアを算出するためのモデルとプログラムです。
ボウリングのスコアのつけ方について簡単に確認しておきましょう。

==== ボウリングの基本ルール

ボウリングは10フレームで構成されます。各フレームでは、プレイヤーは最大2回投球できます。

* 1投目で10本すべてのピンを倒した場合、「ストライク」となり、そのフレームは終了
* 2投で10本すべてのピンを倒した場合は「スペア」
* 2投しても10本倒せなかった場合は「ミス」

==== スコアの計算方法

各フレームのスコアは、倒したピン数だけでなく、ストライクやスペアの場合はボーナス点が加算されます。

ストライクの場合::
* 10点 + 次の2投で倒したピン数がボーナス点として加算される
* 例: ストライク（10点）+ 次の1投目（7点）+ 次の2投目（2点）= 19点

スペアの場合::
* 10点 + 次の1投で倒したピン数がボーナス点として加算される
* 例: スペア（5+5=10点）+ 次の1投目（7点）= 17点

ミスの場合::
* 倒したピン数の合計がそのフレームのスコアになる
* ボーナス点はなし

==== 10フレーム目の特別ルール

10フレーム目は特別なルールがあります：

* ストライクまたはスペアの場合、追加で1投または2投できる
* これにより、10フレーム目は最大3投できる可能性がある

==== スコア計算の例

以下のような投球があった場合のスコア計算例です：

.スコア計算例
[cols="1,3,2,2"]
|===
|フレーム |投球結果 |フレームスコア |累計スコア

|1
|7, 0
|7
|7

|2
|5, 5 (スペア)
|10 + 次の1投(10) = 20
|27

|3
|10 (ストライク)
|10 + 次の2投(10+5) = 25
|52

|4
|10 (ストライク)
|10 + 次の2投(5+4) = 19
|71

|5
|5, 4
|9
|80
|===

この計算ルールをUMLでモデル化し、C言語で実装していくのが、このチュートリアルの目標です。


[NOTE]
--
このように、ボウリングのスコア計算は、単純な足し算ではなく、「ストライクとスペアでは異なる方法で次のフレームの結果を待つ」という状態管理が必要になります。
この複雑さを、UMLのステートマシン図を使ってモデル化し、それをC言語の状態機械として実装してみましょう。
