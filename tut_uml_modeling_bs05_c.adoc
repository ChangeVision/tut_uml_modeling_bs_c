== できあがったプログラムを試す

[.lead]
対応づけルールを使って、モデルから変換して作成したプログラムができあがりました。
できあがったプログラムを確認し、実際に動かしてみましょう。

=== プログラムを完成させる

まず、プログラムをテストできるよう、完成させましょう。


==== 作成したプログラムを確認する

作成したプログラムの全体を確認しましょう。
C言語版では、プログラムは以下の3つのファイルで構成されています：

* `bowling.h` - ヘッダーファイル（構造体と関数の宣言）
* `bowling.c` - 実装ファイル（関数の実装）
* `main.c` - メインプログラム（テストコードを含む）

[[c_header_file]]
.【C言語】bowling.h（ヘッダーファイル）
[source,c]
----
/* bowling.h - ボーリングスコア管理システム */
#ifndef BOWLING_H
#define BOWLING_H

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <stdbool.h>
#include <time.h>

/* 定数定義 */
#define MAX_FRAMES 15
#define MAX_PLAYERS 10
#define MAX_GAMES 20
#define MAX_NAME_LEN 64
#define ID_LEN 12

/* フレームの状態 */
typedef enum {
    RESERVED,
    BEFORE_1ST,
    BEFORE_2ND,
    PENDING,
    FIXED
} FrameState;

/* Frame構造体: 1フレーム分のピン数やボーナスを記録 */
typedef struct {
    int frame_no;
    int first;
    int second;
    int spare_bonus;
    int strike_bonus;
    int total;
    FrameState state;
} Frame;

/* Score構造体: 各人の10フレーム分のスコアを記録 */
typedef struct {
    char id[ID_LEN];
    char player[MAX_NAME_LEN];
    int fno;
    Frame frames[MAX_FRAMES];
    ScoreState state;
} Score;

/* その他の構造体と関数の宣言... */

#endif /* BOWLING_H */

==== テスト用のプログラムを追加する

`main.c` には、すでにテスト用のプログラムが含まれています（ <<c_test_code>> ）。
実際にピン数を与えてスコアを作成しているのは「 `game_playing` 」関数の呼び出しです。
このテストコードでは、ピン数のデータは、1ゲーム分のピン数を配列にまとめています。
しかし、スコアシートのプログラム本体はピン数を受け取るたびに動作するように作られているので、ピン数を1投球ずつ与える方法でも動かせます。

[[c_test_code]]
.【C言語】main.c（テストコード部分）
[source,c]
----
int main(void) {
    /* スコアシートを作成 */ // <1>
    ScoreSheet sheet;
    scoresheet_init(&sheet, time(NULL)); // <2>
    
    /* ゲームを追加 */ // <3>
    scoresheet_add_game(&sheet, 1);
    Game *game = &sheet.games[sheet.game_count - 1]; // <4>
    
    /* プレイヤーをエントリー */ // <5>
    game_entry(game, "くぼあき");
    game_entry(game, "うえはら");
    
    /* スコアシート情報を表示 */ // <6>
    scoresheet_print(&sheet);
    printf("\n");
    
    /* ゲームレコード（実際の投球データ） */ // <7>
    int game_records[2][30] = {
        {7, 0, 5, 5, 10, 10, 5, 4, 10, 7, 3, 5, 4, 7, 3, 7, 3, 4},
        {6, 3, 9, 0, 0, 3, 8, 2, 7, 3, 10, 9, 1, 8, 0, 10, 6, 3}
    };
    int record_index[2] = {0, 0};
    
    /* ゲーム終了まで繰り返し */ // <8>
    while (!game_is_finished(game)) {
        printf("----------------------------------------\n");
        printf("turn: %s\n", game_turn_player_name(game));
        
        int score_index = game->turn; // <9>
        int pins = game_records[score_index][record_index[score_index]]; // <10>
        record_index[score_index]++;
        
        printf("input pins: %d\n", pins);
        game_playing(game, score_index, pins); // <11>
        
        game_print(game); // <12>
        
        /* Enterキーで次へ進む */ // <13>
        printf("Press Enter to continue...");
        getchar();
    }
    
    printf("Game(id:%s) is finished.\n", game->id);
    
    return 0;
}
----
<1> 新しいスコアシートを作成する準備
<2> スコアシートを初期化（現在時刻を記録）
<3> スコアシートにゲームを追加
<4> 追加したゲームを選ぶ（ここでは最後のゲームを選ぶ方法をとった）
<5> ゲームへプレーヤーがエントリーする。このプレーヤーのこのゲーム用のスコアが用意される
<6> スコアシートの作成日付とIDの表示
<7> テスト用データ。1行が1人の1ゲーム分のピン数のデータ
<8> ゲームの終了まで繰り返し
<9> `turn`は、そのゲーム内の何番目のプレーヤーか（ゲーム中の何番目のスコアか）を表している。このテストコードでは2人がエントリしているので、0→1→0→1 と繰り返す
<10> 現在のturnのプレーヤーのピン数のテストデータを先頭から1つずつ取り出す
<11> 現在のturnのプレーヤーのスコアにピン数を渡して、スコアへ反映する
<12> 現在のゲームの全体像を取得して表示する
<13> ここで空の改行キーを入力してテストを進める。テストを1回ずつ止めて実行できる。この入力待ちをやめれば一括して実行できる

<1> Makefileを使ってビルド。または手動で `gcc` コマンドを実行してもよい

NOTE: Windowsの場合、MinGW-w64またはVisual Studio Build Toolsが必要です。macOSやLinuxの場合は、gccまたはclangがインストールされていれば動作します。

=== C言語版とRuby版の違い

C言語版では、以下の点がRuby版と異なります：

==== ID生成

Ruby版では `SecureRandom.urlsafe_base64` を使用していましたが、C言語版ではシンプルなカウンターベースのID生成を使用しています。

[source,c]
----
void generate_id(char *id) {
    static int counter = 0;
    snprintf(id, ID_LEN, "ID%08d", counter++);
}
----

==== メモリ管理

Ruby版では配列のサイズを動的に拡張できましたが、C言語版では固定サイズの配列を使用しています。
これにより、メモリ管理がシンプルになり、メモリリークのリスクが削減されます。

==== 状態の表現

Ruby版ではシンボル（`:RESERVED`など）を使用していましたが、C言語版では列挙型（`enum`）を使用しています。

[source,c]
----
typedef enum {
    RESERVED,
    BEFORE_1ST,
    BEFORE_2ND,
    PENDING,
    FIXED
} FrameState;
----
